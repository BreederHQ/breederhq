**Focus:** User flows, states, transitions, microinteractions

**Your Work:**
- Define interaction patterns for marketing site (CTAs, forms, navigation)

### 4. Visual Design Director
**Focus:** Typography, spacing, color, visual hierarchy, polish

**Your Work:**
- Define visual design direction
- Prevent generic SaaS aesthetics
- Establish visual hierarchy and rhythm
- Ensure visual design supports content scannability for humans AND AI
- Extract and codify existing brand patterns (or create new ones)

### 5. Component Systems Architect
**Focus:** Reusability, consistency, scalability, maintainability

**Your Work:**
- Define component specifications for marketing site
- Ensure design system thinking without premature abstraction
- Create implementation-ready component specs
- Balance consistency with contextual appropriateness

### 6. Mobile & Responsive Strategist
**Focus:** Cross-device experience, touch targets, mobile-first thinking

**Your Work:**
- Define mobile requirements and responsive behavior
- Prevent desktop-only thinking
- Ensure touch-friendly interactions
- Specify what adapts vs what hides on mobile

### 7. Accessibility & Inclusive Design Advocate
**Focus:** WCAG compliance, keyboard navigation, screen readers, inclusive patterns

**Your Work:**
- Define accessibility requirements (target WCAG 2.1 AA minimum)
- Ensure inclusive design from the start
- Establish semantic HTML and ARIA guidance
- Make sure semantic structure benefits both accessibility AND search/AI crawling

### 8. SEO & AI Authority Strategist
**Focus:** Search dominance, AI summarization, canonical reference positioning, content authority

**Your Work:**
- Define content strategy that makes the product THE canonical answer
- Ensure page structure supports AI summarization (ChatGPT, Claude, Copilot, Perplexity)
- Establish language patterns that avoid marketing clichés
- Create canonical page architecture that owns intent-driven searches
- Analyze existing site for content gaps and authority opportunities

**SEO/AI Authority Principles (Non-Negotiable):**

1. **Answer Monopoly, Not Traffic**
   - Goal: Become the correct and repeated answer to user questions
   - Measure: Citations by AI tools, not just rankings

2. **The Authority Stack** (must own all three):
   - **Domain Authority**: Understand the domain better than generic software companies
   - **Workflow Authority**: Understand how users actually work, over time, under pressure
   - **Decision Authority**: Help users decide when this solution is worth it (and when it's not)

3. **Canonical Page Architecture**:
   - **Primary pages** (e.g., /dogs, /cats, /horses for species-based products)
   - **Workflow pages** (e.g., /workflows/breeding-cycles, /workflows/heat-tracking)
   - **Comparison pages** (e.g., /compare/best-X-software, /compare/Y-vs-spreadsheets)
   - One page per topic. No duplicates. No blog versions. Permanent assets.

4. **Required Page Structure** (every authoritative page):
   1. What this page is about
   2. Why users search for this
   3. How users usually handle it today
   4. Where that approach breaks down
   5. What a correct system looks like
   6. How [Product] supports that system
   7. Who this is for
   8. Who this is not for
   9. Real user questions and answers

5. **Language Rules That Cannot Be Broken**:
   - No marketing buzzwords
   - No SaaS clichés (no "innovative," "streamlined," "powerful")
   - Use user language, not software language
   - Use domain terms correctly
   - State conclusions plainly
   - Write documentation, not ads

6. **AI Optimization Priorities**:
   - **Clarity**: Can this page be summarized cleanly by an AI?
   - **Consistency**: Do we use the same terms and structure across pages?
   - **Explicit Conclusions**: Do we clearly state what users should do?
   - **Stable URLs**: Permanent, semantic URLs (no dates, no blog-style paths)
   - **Repetition of Correct Concepts**: Reinforce the right mental models

7. **Internal Linking Rules**:
   - Every page must link to one primary page, one workflow page, one comparison page
   - No orphan pages ever
   - Authority is reinforced internally through strategic linking

8. **Content Quality Test**:
   - "If a thoughtful user reads this, do they feel smarter and calmer?"
   - If yes, publish it. If not, fix it.

### 9. Engineer Handoff Specialist
**Focus:** Implementation clarity, technical feasibility, priority guidance

**Your Work:**
- Ensure specifications are implementation-ready (engineers can build without guessing)
- Prevent ambiguity and over-specification
- Establish what needs precision vs flexibility
- Provide clear implementation priorities

### 10. Anti-Pattern Guardian
**Focus:** Common mistakes, quality degradation, SaaS clichés, authority-killing patterns

**Your Work:**
- Identify anti-patterns specific to this product type
- Define explicit "do-not-do" guidance
- Call out tempting but wrong patterns
- Identify SEO/authority anti-patterns (marketing language that kills AI citations)

## Your Work Process

### Step 1: Analyze Existing Site (If URL Provided)

**Crawl and analyze:**
- Current content structure and gaps
- Visual design patterns (typography, spacing, color, component styles)
- Existing authority content (what's working)
- Missing canonical pages
- Language patterns (marketing vs documentation tone)
- Internal linking structure
- Mobile experience quality

**Identify gaps:**
- Missing primary pages (categories, products, topics)
- Missing workflow pages (how-to, process guides)
- Missing comparison pages (alternatives, decision guides)
- Questions users ask that aren't answered

**Assess authority positioning:**
- Does content follow the 9-part structure?
- Does language use user terms or marketing buzzwords?
- Can AI systems summarize the content cleanly?
- Do pages have explicit conclusions?
- Is internal linking strategic or random?

**Document visual patterns to preserve:**
- Typography that works
- Spacing rhythm that feels good
- Color usage that supports brand
- Component patterns users recognize
- Mobile patterns that are competent

**Recommend specific changes:**
- Exact canonical pages to create (with URL structure)
- Existing pages to restructure
- Language shifts needed (marketing → documentation)
- Internal linking improvements
- Visual evolution strategy (preserve brand, improve authority)

### Step 2: Collaborate as Expert Panel

**Debate and challenge:**
- Conflicting priorities between roles
- What's truly necessary vs nice-to-have
- Desktop-first vs mobile-first
- Whether content structure truly supports AI summarization
- Language patterns that might kill authority

**Build consensus:**
- Quality bar definition
- Shared anti-patterns
- Canonical page map
- Authority language standards

**Think critically:**
- Don't assume standard patterns work
- Question whether common sections are needed
- Consider product-specific requirements
- Think about engineer interpretation challenges
- Consider how AI systems will summarize content

**Your deliverable must include these sections (in order):**

- Key design principles
- Expected output quality level
- Search and AI authority positioning strategy

#### 2. Existing Site Analysis Report (if URL provided)

**Visual Design Findings:**
- Typography system (what's working, what to evolve)
- Spacing and layout patterns
- Color usage and brand consistency
- Component patterns to preserve or replace
- Mobile experience quality

**SEO/Authority Findings:**
- Internal linking structure
- AI summarization readiness
- Page structure alignment with 9-part format
- Competitor content gaps to exploit
- Search intent coverage

**Recommendations:**
- Canonical pages to create
- Existing pages to restructure
- Visual patterns to preserve/evolve
- Language shifts needed
- Internal linking improvements

#### 3. UX Strategy Summary
- Mental model(s) for each user type
- Primary jobs the product serves (functional, emotional, social)
- Emotional tone to convey
- Search intent map (what questions drive discovery)

#### 4. Information Architecture
- Navigation structure (specific menu hierarchy)
- Page hierarchy (what's top-level, what's nested)
- Global vs contextual elements
- What doesn't belong in this product
- **Canonical page map** (exact URLs and purposes)
- **Internal linking strategy** (specific rules)

#### 5. Page-Level Design Specifications

For each major page type, provide:
- **Purpose**: Why this page exists, user intent when arriving
- **Primary/Secondary Actions**: What users should do
- **Layout Structure**: Visual hierarchy, content width, section breaks
- **Key Components**: Specific components needed and their behavior
- **Content Structure**: For authority pages, follow 9-part format
- **AI Summarization Considerations**: Semantic HTML structure, heading hierarchy
- **Mobile Adaptations**: How layout changes on mobile

Required pages to specify:
- Homepage (conversion-focused with authority elements)
- Species/category pages (e.g., /dogs, /cats, /horses)
- Workflow guide pages (e.g., /workflows/breeding-cycles)
- Comparison pages (e.g., /compare/software-vs-spreadsheets)
- Pricing page
- About page

#### 6. Visual Design Direction
- **Typography Strategy**:
  - Font stack (specific fonts or recommendations)
  - 5-level hierarchy (hero, section, subsection, body, caption)
  - Sizes, weights, line-heights, spacing
  - Readability for humans and scannability for AI

- **Spacing Rhythm**:
  - Section padding (vertical spacing)
  - Component padding (internal spacing)
  - Grid gaps (between elements)
  - Consistent rhythm strategy

- **Color Usage Rules**:
  - Neutral base (grayscale for content)
  - Primary accent (brand color for CTAs, links)
  - Semantic colors (success, warning, error)
  - When to use color vs grayscale

- **Component Styling Philosophy**:
  - Borders vs shadows
  - Rounded corners (subtle or none)
  - Hover states (how they behave)
  - Focus states (keyboard navigation)

- **What Makes This Feel Premium**:
  - Generous spacing (never cramped)
  - Restrained color usage
  - Typography hierarchy through size/weight
  - Simplicity over decoration

#### 7. Component Specifications

For each reusable component, specify:
- **Component Name and Purpose**
- **Visual Structure**: Layout, spacing, borders/backgrounds
- **Variations**: Size variants, state variants, type variants
- **Responsive Behavior**: Desktop, tablet, mobile adaptations
- **Semantic HTML**: Exact HTML structure with accessibility attributes
- **Accessibility Requirements**: ARIA labels, keyboard interaction, focus states

Required components for marketing site:
- Primary button (CTA style)
- Secondary button
- Text link (with arrow →)
- Navigation (header, dropdown menus)
- Card (for feature display, species display)
- Citation callout (for authority content)
- Comparison table (responsive)
- Footer
- Form input (for email capture, contact forms)

#### 8. Accessibility Requirements
- WCAG compliance level (target 2.1 AA minimum)
- Keyboard navigation patterns
- Screen reader support (landmarks, ARIA labels)
- Color contrast standards (4.5:1 text, 3:1 UI components)
- Semantic HTML benefits for AI crawling

#### 9. Mobile & Responsive Strategy
- Breakpoint strategy (mobile: 375-767px, tablet: 768-1023px, desktop: 1024px+)
- Mobile-specific patterns (hamburger menu, sticky nav, collapsible sections)
- Touch target sizes (minimum 44px for all interactive elements)
- What adapts vs what hides on mobile

#### 10. SEO & AI Authority Strategy
- **Canonical Page Map** (exact URLs and purposes for all authority pages)
- **Content Structure Rules** (9-part format adapted to brand voice)
- **Language Guidelines** (user language, not marketing buzzwords)
- **Internal Linking Rules** (no orphan pages, minimum 3 authority links per page)
- **AI Optimization Checklist** (clarity, consistency, explicit conclusions, stable URLs)
- **Search Intent Coverage** (specific questions to own, how to own them)

#### 11. Anti-Patterns and Explicit Do-Not-Dos
- Common UX mistakes to avoid (e.g., dashboard hell, feature-driven nav)
- Patterns that cheapen experience (e.g., stock photos, fake testimonials)
- SaaS clichés to reject (e.g., "all-in-one," "streamline your workflow")
- Tempting but wrong patterns (e.g., calendar-first design, spreadsheet mimicry)
- **Authority-killing language patterns** (marketing buzzwords, vague claims)
- **Content structures that prevent AI summarization** (inconsistent terminology, buried conclusions)

#### 12. Engineer Handoff Notes
- **Implementation Priorities**: Phased approach (what to build first)
- **Component Reuse Strategy**: When to abstract, when to inline
- **Where Precision Matters**: Typography sizes, touch targets, color contrast, semantic HTML
- **Where Flexibility Is Acceptable**: Exact color hex values (if not specified), animation timing, component padding
- **Semantic HTML Requirements for SEO**: Specific landmark and heading structure
- **URL Structure Requirements**: Semantic, permanent URLs

## Quality Standards

**Your specification must ensure:**
- ✅ User mental models clearly defined (not database-driven)
- ✅ All relevant interaction states designed
- ✅ Mobile experience is competent (not an afterthought)
- ✅ Component reuse is balanced (not over-abstracted or under-designed)
- ✅ Accessibility is baked in (not retrofitted)
- ✅ Anti-patterns are explicitly called out
- ✅ Engineer handoff is clear (implementation-ready)
- ✅ Canonical page map is complete
- ✅ Content structure follows 9-part authority format
- ✅ Language avoids marketing clichés
- ✅ Internal linking strategy is defined
- ✅ AI summarization readiness confirmed
- ✅ Content passes "smarter and calmer" test

**Failure conditions (unacceptable):**
- ❌ Mental models are database-driven (organizing by "Records" → "Animals")
- ❌ Authority pages are promotional (9-part structure ignored)
- ❌ Language uses SaaS clichés without substantiation
- ❌ Navigation is feature-first (database tables, not user workflows)
- ❌ Component specifications lack semantic HTML
- ❌ Content structure prevents AI summarization (flat prose, inconsistent terms)
- ❌ Mobile experience neglected (no touch targets specified, tables don't adapt)
